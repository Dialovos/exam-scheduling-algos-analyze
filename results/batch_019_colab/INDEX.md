# batch_019_colab — Phase-2 validation

Minimal batch: Phase-2 cached/Thompson variants across ITC 2007 sets 1–8.
Generated on Colab (A100 runtime), flattened + polished on 2026-04-24.

## Layout

```
batch_019_colab/
├── INDEX.md                     this file
├── summary.csv                  120 runs: (algo, instance, seed, feasible, hard, soft, runtime_sec, iterations)
└── <algo>_<set>_<seed>/         120 per-run dirs (result.json, stderr.log, solutions/*.sln)
```

## Scope (5 algos × 8 ITC sets × 3 seeds = 120 runs)

| Algorithm | Notes |
|---|---|
| `tabu_cached` | Phase-2 cached Tabu (Ecach.move_delta), 3000 iters |
| `sa_cached` | Phase-2 cached SA |
| `gd_cached` | Phase-2 cached Great Deluge |
| `lahc_cached` | Phase-2 cached LAHC |
| `alns_thompson` | ALNS with Thompson-sampling AOS |

Seeds: 42, 43, 44 — three per (algo, instance).

## What's intentionally missing vs `batch_018_colab`

This batch is a **focused validation run** — not the full paper matrix. Specifically NOT included here:

| Missing | Present in `batch_018_colab` |
|---|---|
| Chain-tournament tuning | `colab_batch_chain/` (top-5 chains + checkpoints) |
| Scaling ladder (n=50..1000) | `colab_batch_scaling/` |
| Parameter sensitivity sweep | `colab_batch_sweep/` |
| CP-SAT / IP runs | `colab_batch_ip/` |
| Non-cached algorithms | `greedy/tabu/sa/alns/...` (13 algos total) |
| Extended seed count | 7 seeds per (algo, set) |

If you need any of the above, re-run `notebooks/colab_runner.ipynb` which covers the full matrix.

## Purpose

Validate that the Phase-2 cached-evaluator variants (`*_cached`, `alns_thompson`)
produce correct solutions on the full ITC 2007 set. Primary acceptance criterion:

- `feasible == true` on sets 1–8
- `soft` within the expected ballpark of the non-cached parent from `batch_018_colab`
- Phase-2 speedup: cached variants run 3–12× faster than their non-cached parents
  (measured separately in `docs/PERF_ROADMAP.md`)
